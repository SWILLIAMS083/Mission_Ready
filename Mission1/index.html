HTML Requirements (What I can explain)

These are the exact elements required for Mission 1, and where they appear on this page.

  • <head> — holds metadata + the CSS link (top of this file).
  • <body> — wraps everything visible on the page.
  • <div> — used as containers for layout (ex: .container, .card).
  • <h1> — main heading (my name at the top of the page).
  • <p> — paragraph text (tagline + intros + descriptions).
  • <img> — image used once in the header (logo) to demonstrate the tag.
  • <a> — links (nav links jump to sections; email link in contact section).
  • Class — reusable styling (ex: class="card" used multiple times).
  • ID — unique targeting (ex: id="contact" is one section only).
  • Email Contact Form — the form below uses mailto: for simplicity.

CSS Requirements (What I can show)

These are the exact CSS requirements and where they appear in styles.css.

  • Flexbox — used in .nav, .cards, and .skills for layout.
  • Background stylisation — body uses a gradient background.
  • Font stylisation — font-family and font sizes are set on headings/text.
  • Margins — spacing outside elements (ex: headings and paragraphs).
  • Width & Height — cards and the header logo have sizing rules (ex: .card, .logo).
  • Animations — the button uses a @keyframes animation called pulse.
